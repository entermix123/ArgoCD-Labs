We need to have working Kubernetes cluster on Linux OS



1. INSTALL ARGOCD CLI ON LINUX
==============================


Download ArgoCD
	terminal --> curl -sSL -o argocd-linux-amd64 https://github.com/argoproj/argo-cd/releases/latest/download/argocd-linux-amd64

Install ArgoCD
	terminal --> sudo install -m 555 argocd-linux-amd64 /usr/local/bin/argocd

Check if ArgoCD CLI is installed
	terminal --> argocd version --client

Find the initial password
	terminal --> kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d

	# result: password

In order to interact with Argo CD using CLI, Argo CD server need to be reached. We need to find argocd server ip address and port.

Find node01 IP addresses
	terminal --> k get nodes -o wide

# example result:
NAME           STATUS   ROLES           AGE   VERSION   INTERNAL-IP   EXTERNAL-IP   OS-IMAGE             KERNEL-VERSION
controlplane   Ready    control-plane   21d   v1.34.1   172.30.1.2            Ubuntu 24.04.3 LTS   ...
node01         Ready              	21d   v1.34.1   172.30.2.2            	Ubuntu 24.04.3 LTS   ...

We need column INTERNAL-IP of node01 - 172.30.2.2


Find Argocd service ports
	terminal --> k get svc -n argocd

# example result:
NAME                                      TYPE        CLUSTER-IP       EXTERNAL-IP   PORT(S)                      AGE
argocd-applicationset-controller          ClusterIP   10.98.122.13     <none>        7000/TCP,8080/TCP            48m
argocd-dex-server                         ClusterIP   10.97.96.48      <none>        5556/TCP,5557/TCP,5558/TCP   48m
argocd-metrics                            ClusterIP   10.106.247.183   <none>        8082/TCP                     48m
argocd-notifications-controller-metrics   ClusterIP   10.105.221.222   <none>        9001/TCP                     48m
argocd-redis                              ClusterIP   10.102.9.157     <none>        6379/TCP                     48m
argocd-repo-server                        ClusterIP   10.100.149.16    <none>        8081/TCP,8084/TCP            48m
argocd-server                             NodePort    10.102.163.27    <none>        80:32073/TCP,443:30164/TCP   48m	# target
argocd-server-metrics                     ClusterIP   10.102.10.150    <none>        8083/TCP                     48m

We will use 32073 port (from 80:32073) 


Login to Argo CD using CLI, use argocd login command.
For HTTPS (default - TLS enabled):
	terminal --> argocd login 172.30.2.2:30164 --username admin --password <password> --grpc-web

For HTTP (if running insecurely):
	terminal --> argocd login <node-ip>:<http-nodeport> --username admin --password <password> --insecure --grpc-web

	# if we have TLS enabled we don't have to use '--insecure'
	# result:
	'admin:login' logged in successfully
	Context '172.30.2.2:32073' updated


Verify logging into ArgoCD server
	terminal --> argocd cluster list --grpc-web

# result:
SERVER                          NAME        VERSION  STATUS   MESSAGE                                                  PROJECT
https://kubernetes.default.svc  in-cluster           Unknown  Cluster has no applications and is not being monitored.  





2. Install Argo-rollouts controller and kubectl-argo-rollouts (kargo) on Linux
==============================================================================

Installation documentation - https://argo-rollouts.readthedocs.io/en/stable/installation/


1. Install Argo Rollouts Controller
-----------------------------------

install.yaml - Standard installation method.

Create argo-rollouts namespace (required)
	terminal --> kubectl create namespace argo-rollouts

	# result: namespace/argo-rollouts created

Apply the latest stable manifest (installs CRDs, controller, RBAC, etc.)
	terminal --> kubectl apply -n argo-rollouts -f https://github.com/argoproj/argo-rollouts/releases/latest/download/install.yaml

	# result:
	customresourcedefinition.apiextensions.k8s.io/analysisruns.argoproj.io created
	customresourcedefinition.apiextensions.k8s.io/analysistemplates.argoproj.io created
	customresourcedefinition.apiextensions.k8s.io/clusteranalysistemplates.argoproj.io created
	customresourcedefinition.apiextensions.k8s.io/experiments.argoproj.io created
	customresourcedefinition.apiextensions.k8s.io/rollouts.argoproj.io created
	serviceaccount/argo-rollouts created
	clusterrole.rbac.authorization.k8s.io/argo-rollouts created
	clusterrole.rbac.authorization.k8s.io/argo-rollouts-aggregate-to-admin created
	clusterrole.rbac.authorization.k8s.io/argo-rollouts-aggregate-to-edit created
	clusterrole.rbac.authorization.k8s.io/argo-rollouts-aggregate-to-view created
	clusterrolebinding.rbac.authorization.k8s.io/argo-rollouts created
	configmap/argo-rollouts-config created
	secret/argo-rollouts-notification-secret created
	service/argo-rollouts-metrics created
	deployment.apps/argo-rollouts created

Verify controller is running
	terminal --> kubectl get pods -n argo-rollouts
# Expected: argo-rollouts deployment with 1/1 ready pod



2. Install kubectl-argo-rollouts (kargo)
----------------------------------------

Download the latest Linux amd64 binary
	terminal --> curl -LO https://github.com/argoproj/argo-rollouts/releases/latest/download/kubectl-argo-rollouts-linux-amd64

Make it executable
	terminal --> chmod +x kubectl-argo-rollouts-linux-amd64

Move to a directory in your PATH (requires sudo)
	terminal --> sudo mv kubectl-argo-rollouts-linux-amd64 /usr/local/bin/kubectl-argo-rollouts

Verify Installation
	terminal --> kubectl argo rollouts version
	
	# Expected: Shows plugin version (e.g., kubectl-argo-rollouts: v1.8.x...)


Start the Dashboard (uses port-forwarding):
-------------------------------------------
Global (searches all namespaces for Rollouts)
	terminal --> kubectl argo rollouts dashboard

For scoped to a specific namespace (recommended if Rollouts are namespaced)
	terminal --> kubectl argo rollouts dashboard -n <target-namespace>

	# Example: kubectl argo rollouts dashboard -n argo-rollouts

# Access in browser: http://localhost:3100
# Stop with Ctrl+C
